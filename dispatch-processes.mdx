## **3.1 Dispatch Workflow Overview**

| Stage | Timing / Action |
| :-- | :-- |
| Booking Confirmed | – |
| Assign Driver | T-24 Hours before trip |
| Send Driver Trip Details | T-24 Hours |
| Send Customer Chauffeur Details | T-24 Hours |
| Driver Confirms Readiness | T-2 Hours |
| Driver En Route | T-60 Minutes |
| Driver Marks Arrival | On Location |
| Passenger On Board | Trip Begins |
| Drop-Off | Trip Completed |
| Post-Trip | Close ride in system |

💡 **Info:** Hourly, Sprinter, BUS, and Limo rides must have **dispatch prepared 7 days before the trip**.

## **3.2 Driver Assignment Process**

### **24 Hours Before Trip – Review Booking Details**

| Field | Notes |
| :-- | :-- |
| Pickup time & location | Verify against booking |
| Drop-off location | Confirm full address |
| Vehicle type required | Ensure correct assignment |
| Special requests | Car seats, accessibility, beverages, signage |
| Customer notes | Any additional information |

### **Driver Selection Criteria**

| Criteria | Notes |
| :-- | :-- |
| Driver availability | Confirm free schedule |
| Vehicle match | Ensure correct type & capacity |
| Driver location/proximity | Choose nearest driver |
| Driver ratings/experience | Prioritize VIP clients |
| Special skills | Airport protocol, wheelchair accessible, etc. |

### **Assign in Portal**

| Step | Action |
| :-- | :-- |
| 1 | Select driver from available list |
| 2 | Attach trip to driver's schedule |
| 3 | Confirm driver acceptance |
| 4 | Send trip details to driver (include link) |

**Trip Details to Include:**

- All trip details
- Customer contact info
- Special instructions
- Route suggestions (if applicable)

## **3.5 Day-of-Trip Monitoring**

| Time | Dispatch Action |
| :-- | :-- |
| T-2 Hours | Confirm driver is awake and preparing |
| T-1 Hour | Verify vehicle is ready |
| T-1 Hour | Confirm driver is en route to pickup |
| Pickup Time | Verify driver arrived and marked in system |
| On Board | Confirm passenger picked up |
| Drop-Off | Confirm trip completed |

💡 **Info:** Large vehicles must be checked at least 5 hours before departure; take a **vehicle check video from driver** (especially for buses).

**Dispatch Check-In Script (Call to Driver):**

> "Hi \[Driver Name\], this is dispatch confirming you're good for the \[Customer Name\] pickup at \[Time\]. Everything on track?"

## **3.6 Real-Time Issue Management**

| Issue | Resolution |
| :-- | :-- |
| Driver running late | Call customer immediately, provide new ETA, offer discount if \>15 min delay |
| Flight delay | Monitor flight tracker, notify driver, update customer |
| Vehicle breakdown | Dispatch backup vehicle, update customer, assign new driver if needed |
| Customer no-show | Driver waits 15 min, calls customer twice; dispatch calls customer, document, charge per policy |
| Wrong address | Confirm correct address, update driver, adjust route |
| Driver no-show | Escalate immediately, dispatch backup, call customer, apologize, offer compensation |

**Customer Communication for Delays:**

> "Hi \[Customer Name\], this is \[Company Name\] dispatch. Your chauffeur is approximately \[X\] minutes behind due to \[reason\]. New arrival time: \[Time\]. I sincerely apologize for any inconvenience. Is there anything I can do to help in the meantime?"

## **3.7 Trip Closure Process**

**Steps After Every Ride:**

| Step | Action |
| :-- | :-- |
| 1 | Driver marks trip complete in app/portal |
| 2 | Dispatch reviews for issues or notes |
| 3 | Close trip in system including: <br /> - Actual pickup/drop-off times <br /> - Mileage (if applicable) <br /> - Driver notes <br /> - Any incidents |
